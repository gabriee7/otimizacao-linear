Teoria de Otimização Linear

1. Definição e Estrutura de um PL

Um Problema de Programação Linear (PL) consiste em otimizar (maximizar ou minimizar) uma função linear sujeita a restrições lineares de igualdade ou desigualdade.

Elementos de um PL

Variáveis de decisão: x₁, x₂, …, xₙ — as incógnitas do problema.

Função objetivo: f(x) = cTx que se deseja max ou min.

Restrições: Ax ≤ b (ou ≥ b, ou = b) que limitam as soluções viáveis.

Não-negatividade: xⱼ ≥ 0 para todo j (na maioria dos modelos práticos).

2. Forma Padrão

O método Simplex opera na forma padrão, em que todas as restrições são igualdades e as variáveis são não-negativas.

Regra de conversão: Uma restrição do tipo ≤ vira igualdade adicionando uma variável de folga s ≥ 0. Uma restrição ≥ vira igualdade subtraindo uma variável de excesso e adicionando uma variável artificial.
Forma Padrão
Maximizar: z = cTx

Sujeito a: Ax = b
            x ≥ 0

Exemplo de conversão

Restrição original: 2x₁ + 3x₂ ≤ 12

Forma padrão: 2x₁ + 3x₂ + s₁ = 12,   s₁ ≥ 0

3. Forma Canônica e Solução Básica Viável (SBV)

Dado um sistema Ax = b com m restrições e n variáveis (n > m), escolhemos m variáveis básicas que formam uma submatriz B de A invertível. As demais são não-básicas (fixadas em 0).

Solução Básica

Fazendo as variáveis não-básicas = 0, obtemos a solução básica: xB = B-1b.

Se xB ≥ 0, a solução é uma Solução Básica Viável (SBV) — e corresponde a um vértice da região viável.

Teorema fundamental da PL: Se existe solução ótima, ela ocorre em alguma SBV (vértice da região viável). O Simplex percorre os vértices até encontrar o ótimo.

4. Método Simplex (Tableau)

O Simplex é um algoritmo iterativo que parte de uma SBV inicial e se move para SBVs adjacentes de valor objetivo crescente (na maximização), até não haver melhora possível.

Passos do algoritmo

1
Montar o tableau inicial
Linhas: uma para cada restrição + linha da função objetivo (z). Colunas: variáveis de decisão, variáveis de folga, e coluna b (RHS).
2
Verificar otimalidade
Se todos os coeficientes na linha z forem ≥ 0 (na maximização), a solução atual é ótima. Pare.
3
Escolher a variável que entra (pivô coluna)
Selecionar a coluna com o coeficiente mais negativo na linha z (critério de Dantzig).
4
Escolher a variável que sai (pivô linha)
Para cada linha com coeficiente positivo na coluna pivô, calcular a razão bi/aij. A linha com a menor razão positiva é a linha pivô (teste da razão mínima).
5
Operação de pivoteamento
Dividir a linha pivô pelo elemento pivô. Usar operações de linha para zerar todos os outros elementos da coluna pivô. Voltar ao passo 2.
Condições de parada

Solução ótima: todos os coeficientes da linha z são ≥ 0.

Problema ilimitado: coluna pivô tem todos os coeficientes ≤ 0 nas linhas de restrição.

Problema inviável: nenhuma SBV inicial pode ser encontrada (variáveis artificiais na base com valor > 0 ao final da Fase I).

5. Método das Duas Fases

Quando há restrições do tipo ≥ ou =, não há SBV imediata. O método das Duas Fases resolve isso criando variáveis artificiais e rodando o simplex em duas rodadas:

Método das Duas Fases

Fase I: Minimizar a soma das variáveis artificiais. Se o mínimo = 0, temos uma SBV inicial viável para o PL original. Caso contrário, o problema é inviável.

Fase II: Usar a SBV final da Fase I como ponto de partida para resolver o PL original.

6. Dualidade

Todo PL (primal) tem associado um problema dual. As soluções dos dois problemas estão intimamente relacionadas pelo Teorema da Dualidade Forte.

Construção do Dual
Primal (Max): Dual (Min):
Max cTx ↔ Min bTy
Ax ≤ b ↔ ATy ≥ c
x ≥ 0 ↔ y ≥ 0

Teoremas de dualidade

D1
Dualidade fraca: Para qualquer x viável do primal e y viável do dual: cTx ≤ bTy.
D2
Dualidade forte: Se o primal tem solução ótima x*, então o dual tem solução ótima y*, e cTx* = bTy*.
D3
Folgas complementares: Na solução ótima, para cada restrição: ou a restrição é ativa, ou a variável dual correspondente é zero.

7. Análise de Sensibilidade

Após encontrar a solução ótima, a análise de sensibilidade determina em que intervalo os parâmetros do problema (coeficientes da função objetivo e RHS) podem variar sem que a base ótima mude.

Variação nos coeficientes da FO (cⱼ)

Para uma variável básica xj: há um intervalo [cjmin, cjmax] dentro do qual a mesma base permanece ótima.

Para uma variável não-básica: cj pode diminuir em no máximo o valor do custo reduzido correspondente sem que ela entre na base.

Variação no RHS (bᵢ)

O valor da FO muda de acordo com os preços sombra (variáveis duais y*): Δz = yi* × Δbi.

Os preços sombra são válidos enquanto bi varia em um intervalo que mantém xB = B-1b ≥ 0.

Dica: Os preços sombra aparecem automaticamente na linha z do tableau final, nas colunas das variáveis de folga originais.
