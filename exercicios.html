R1 Leitura e Extração de Dados

Problema de minimização com 3 variáveis de decisão (x₁, x₂, x₃), todas não-negativas, e 2 restrições do tipo ≥. Nenhuma variável irrestrita em sinal.

Observação importante: x₃ aparece nas restrições com coeficiente 1 em ambas, mas não aparece na função objetivo — seu custo é zero. Isso terá impacto direto na solução.

R2 Classificação — Por que Dual Simplex?

O Dual Simplex é aplicável quando o tableau satisfaz simultaneamente duas condições:

CondiçãoSignificadoEste problema
Dual viávelTodos os valores da linha Z são ≤ 0 (negativos ou zero) — convenção Z − cx = 0✓ satisfaz
Primal inviávelExiste pelo menos um b < 0 na coluna RHS✓ satisfaz
📌 Por que não Simplex Primal? Restrições ≥ geram b negativos após pré-processamento. O Simplex Primal exige todos os b ≥ 0 para iniciar — condição não satisfeita sem variáveis artificiais.
R3 Pré-processamento

3.1 — Verificar se é necessário converter para MIN

O problema já está na forma de minimização, portanto nenhuma conversão é necessária. Igualamos a FO a zero passando tudo para a esquerda com sinal trocado:

MIN Z = 6x₁ + 3x₂  ⟹  Z − 6x₁ − 3x₂ = 0
→ linha Z guarda os coeficientes negativos da FO: [−6, −3, 0, 0, 0] para [x₁, x₂, x₃, x₄, x₅]
⚠️ Atenção: x₃ não aparece na função objetivo — seu custo reduzido inicial é 0. Isso criará um pivô degenerado na iteração 1 (razão = 0/1 = 0), discutido em R5.

3.2 — Adicionar variáveis de excesso e preparar a base

Restrições ≥ → subtraímos variáveis de excesso x₄ e x₅. Uma variável de excesso com coeficiente −1 não pode entrar na base pois geraria valor negativo. Multiplicamos cada restrição por −1 para obter +1 na variável de excesso:

R1:  6x₁ − 3x₂ + x₃ − x₄ = 2   ×(−1)  →   −6x₁ + 3x₂ − x₃ + x₄ = −2
R2:  3x₁ + 4x₂ + x₃ − x₅ = 5   ×(−1)  →   −3x₁ − 4x₂ − x₃ + x₅ = −5

x₄ e x₅ entram na base com b = −2 e −5 — ambos negativos → primal inviável, condição necessária para o Dual Simplex.

3.3 — Verificar viabilidade dual

Na convenção Z − cx = 0, a linha Z guarda −c (negativo dos custos). Para MIN com Dual Simplex, a linha Z deve ter todos os valores ≤ 0 (negativos ou zero):

Variável não-básicaValor na linha Z (= −c)Condição ≤ 0Observação

✅ Pronto para iterar: Dual viável (todos os valores da linha Z ≤ 0) + Primal inviável (b₁ = −2, b₂ = −5) → Dual Simplex aplicável.

O que é Branch-and-Bound?
Ideia simples: Resolver um problema inteiro diretamente é difícil. O B&B resolve versões LP (sem restrições inteiras), e quando aparece variável fracionária — que deveria ser inteira — divide em dois subproblemas: um com a variável "arredondada para baixo" e outro "arredondada para cima".
PassoO que fazer
1. Relaxação LPIgnore as restrições de inteireza. Resolva como LP normal.
2. VerificarSe todas as variáveis inteiras têm valor inteiro → solução ótima!
3. BranchEscolha uma variável inteira com valor fracionário, por ex. x₁=3,5. Crie: Ramo A: x₁≤3 · Ramo B: x₁≥4
4. PodarSe o ramo for inviável ou pior que a melhor solução já encontrada, descarte.
5. RepetirAplique o mesmo processo em cada ramo até encontrar o ótimo inteiro.
